# XGBoost 训练计划与论文答辩要点

## 1. 课题定位先说清楚

当前最合理、最稳妥的课题表述是：

> 基于以太坊地址级交易特征的欺诈风险识别，并结合 SHAP 与业务规则进行欺诈类型倾向分析。

这句话很重要，因为它决定了你后面整套实验、论文和答辩的逻辑边界。

### 为什么要这样定义

因为你现在的数据集标签只有：

- `FLAG = 0`：正常地址
- `FLAG = 1`：欺诈地址

它没有直接给出：

- 钓鱼诈骗与资金清洗
- 庞氏骗局
- 发币骗局与跑路盘

这三类的真实监督标签。

所以你的主任务应该是：

1. **做二分类欺诈风险识别**
2. **做三类欺诈的类型倾向分析**

而不是写成：

> 用监督学习直接训练三分类模型

否则答辩时很容易被问住。

## 2. 整体训练路线图

建议你把整个实验分成 7 个阶段：

1. 数据准备与字段确认
2. 基础探索与任务定义
3. 建立一个简单基线模型
4. 训练 XGBoost 主模型
5. 做模型评估与结果比较
6. 做 SHAP 解释分析
7. 结合三类欺诈规则做类型倾向判断

对应到论文，可以自然写成：

- 数据集与预处理
- 特征体系
- 模型方法
- 实验结果
- 可解释性分析
- 讨论与局限

## 3. 详细训练计划

## 3.1 第一阶段：明确数据和任务

### 目标

先把“你到底拿什么训练”“你到底预测什么”说清楚。

### 当前建议

- 使用清洗后的文件：
  - `processed/transaction_dataset_cleaned.csv`
- 当前数据规模：
  - 9816 条样本
  - 42 个字段
- 标签字段：
  - `FLAG`

### 这一阶段要完成的事情

1. 确认标签只有 `FLAG`
2. 明确本课题主任务是二分类
3. 明确三类欺诈只做“倾向分析”，不写成真实监督三分类

### 论文里怎么写

可以写成：

> 本文首先利用地址级交易统计特征构建欺诈风险识别模型，对地址进行正常 / 欺诈二分类判别；在此基础上，结合三类典型欺诈场景的业务规则与特征贡献，对高风险地址进行类型倾向分析。

## 3.2 第二阶段：确定训练字段

### 不进入基线训练的字段

- `Address`
  - 用来追踪地址，不是行为特征
- `ERC20 most sent token type`
- `ERC20_most_rec_token_type`
  - 这两个文本字段暂时不进入基线训练

### 进入基线训练的字段

基线训练建议先使用 **38 个数值字段**。

### 建议按 5 组理解

#### A. 时间与辅助指示字段

- `has_erc20_activity`
- `Avg min between sent tnx`
- `Avg min between received tnx`
- `Time Diff between first and last (Mins)`

#### B. 交易频次与网络字段

- `Sent tnx`
- `Received Tnx`
- `Number of Created Contracts`
- `Unique Received From Addresses`
- `Unique Sent To Addresses`
- `total transactions (including tnx to create contract)`

#### C. ETH 金额字段

- `min value received`
- `max value received`
- `avg val received`
- `min val sent`
- `max val sent`
- `avg val sent`
- `min value sent to contract`
- `max val sent to contract`
- `avg value sent to contract`
- `total Ether sent`
- `total ether received`
- `total ether sent contracts`
- `total ether balance`

#### D. ERC20 频次与网络字段

- `Total ERC20 tnxs`
- `ERC20 total Ether received`
- `ERC20 total ether sent`
- `ERC20 total Ether sent contract`
- `ERC20 uniq sent addr`
- `ERC20 uniq rec addr`
- `ERC20 uniq rec contract addr`
- `ERC20 uniq sent token name`
- `ERC20 uniq rec token name`

#### E. ERC20 金额字段

- `ERC20 min val rec`
- `ERC20 max val rec`
- `ERC20 avg val rec`
- `ERC20 min val sent`
- `ERC20 max val sent`
- `ERC20 avg val sent`

### 论文里怎么写

可以写成：

> 为保证模型稳定性与可解释性，本文基线实验优先采用清洗后的数值型行为特征进行训练，不将地址标识字段及缺失较多、类别稀疏的文本型代币字段纳入初始模型。

## 3.3 第三阶段：建立简单基线模型

### 为什么要有基线模型

因为论文里不能一上来只给 XGBoost 一个结果。  
你最好告诉老师：

- 简单模型表现如何
- 更复杂模型为什么更好

### 最推荐的基线选择

按本科毕设的工作量，建议只比较 2 到 3 个模型：

1. **Decision Tree**
   - 作用：帮助理解树模型最基本的工作方式
2. **Random Forest**
   - 作用：作为强一点的树模型基线
3. **XGBoost**
   - 作用：作为主模型

如果你不想放太多模型，也可以只保留：

1. Random Forest
2. XGBoost

这样实验不会太乱。

### 这一阶段的目标

不是追求极致精度，而是建立一个“比较参照”。

## 3.4 第四阶段：训练 XGBoost 主模型

### 为什么把 XGBoost 作为主模型

因为它非常适合你的数据类型：

- 数据是表格型结构化数据
- 字段之间存在非线性关系
- 欺诈行为依赖多字段组合
- 样本规模不算特别大
- 训练效率高
- 后续易于和 SHAP 结合

### 第一版参数建议

你不需要一开始就做很复杂的调参。  
第一版可以先从下面这组参数开始：

- `n_estimators = 300`
- `max_depth = 4`
- `learning_rate = 0.05`
- `subsample = 0.8`
- `colsample_bytree = 0.8`
- `eval_metric = 'logloss'`
- `scale_pos_weight ≈ 正常样本数 / 欺诈样本数`

### 为什么先这样设

- 树数 300：足够学习，又不至于太慢
- 深度 4：复杂度适中，比较稳
- 学习率 0.05：学习不算太猛，更稳
- 随机抽样：降低过拟合
- 类别权重：缓解正常 / 欺诈样本不平衡

### 后续如果要调参

只需要先调 4 个最关键参数：

1. `max_depth`
2. `learning_rate`
3. `n_estimators`
4. `scale_pos_weight`

不建议本科阶段做特别庞杂的网格搜索。

## 3.5 第五阶段：训练方式和评估方式

### 数据划分建议

最实用的方案是：

- 训练集：80%
- 测试集：20%
- 使用 `stratify` 保持类别比例一致

如果你想更规范一点：

- 在训练集内部再做 5 折交叉验证

但如果时间紧，先做 80/20 就够了。

### 重点指标

建议至少报告这 5 个指标：

1. Accuracy
2. Precision
3. Recall
4. F1-score
5. AUC

### 在欺诈检测里最重要的是哪几个

优先级建议是：

1. Recall
2. F1-score
3. AUC
4. Accuracy

原因是：

- 欺诈检测里，漏掉真正的欺诈地址会比较危险；
- 所以不能只看 Accuracy。

### 论文里要写出的关键结论

不要只写：

> 模型准确率很高

而要写：

> 在类别不平衡条件下，本文同时关注 Precision、Recall、F1-score 与 AUC 等指标，以更全面评估模型对欺诈地址的识别能力。

## 3.6 第六阶段：SHAP 解释分析

### 为什么一定要做 SHAP

因为你的课题不只是“做一个会预测的模型”，还要让人相信：

- 模型为什么这么判断
- 哪些字段真正重要
- 模型关注的特征和业务逻辑是不是一致

### SHAP 在你的课题里要做两类解释

#### 1. 全局解释

看整体模型最依赖哪些字段。  
例如：

- `Time Diff between first and last (Mins)`
- `avg val received`
- `total ether received`
- `Unique Received From Addresses`
- `Total ERC20 tnxs`

#### 2. 局部解释

看一个具体地址为什么被判成高风险。  
例如某个地址：

- 生命周期极短
- 收款频繁
- 净余额很低

那它就更像钓鱼诈骗与资金快速转移。

### 论文里怎么写

可以写成：

> 为提高模型可解释性，本文引入 SHAP 对训练后的 XGBoost 模型进行全局与局部解释分析，从整体重要特征和单样本贡献两个层面揭示模型的判别依据。

## 3.7 第七阶段：三类欺诈的类型倾向分析

### 这里一定要注意表述

这一步不是：

- 真正的三分类监督学习

而是：

- 基于 SHAP 结果和业务规则的类型倾向分析

### 建议的分析逻辑

#### A. 钓鱼诈骗与资金清洗倾向

重点看：

- `Time Diff between first and last (Mins)` 是否很短
- `Avg min between sent tnx` 是否很短
- `total ether balance` 是否很低

#### B. 庞氏骗局倾向

重点看：

- `Unique Received From Addresses` 是否很高
- `Received Tnx` 是否明显高于 `Sent tnx`
- `avg val received` 是否表现出一定聚集性

#### C. 发币骗局与跑路盘倾向

重点看：

- `Number of Created Contracts` 是否大于 0
- `Total ERC20 tnxs` 是否偏高
- `ERC20 total Ether received` 是否偏高
- `has_erc20_activity` 是否明显

### 最稳妥的写法

你应该写：

> 在欺诈风险识别基础上，本文进一步结合三类典型欺诈场景的业务规则与特征贡献，对高风险地址进行类型倾向分析。

而不要写：

> 本文训练得到了高精度三分类模型

## 5. 论文中必须体现的重点

## 5.1 研究问题要写准

你要体现的是：

- 研究对象是什么
- 数据是什么
- 标签是什么
- 模型解决什么问题
- 解释模块解决什么问题

### 推荐表述

> 本文面向以太坊地址级交易行为数据，构建欺诈风险识别模型，并结合可解释性分析与业务规则实现对典型欺诈类型的倾向判断。

## 5.2 数据清洗要写清楚

必须写出：

- 删除了哪些字段
- 为什么删
- 重复地址如何处理
- ERC20 缺失如何处理
- 文本字段为什么不进入基线模型

这部分已经有清洗报告支持，你后面直接照着写就行。

## 5.3 特征体系要写成“有业务含义的分组”

不要只把 38 个字段机械列出来。  
更好的写法是：

- 时间与生命周期特征
- 交易频次与网络特征
- ETH 金额特征
- 合约交互特征
- ERC20 代币特征

这样老师会觉得你的分析是有逻辑的。

## 5.4 模型选择理由要写出来

为什么不是神经网络？  
为什么不是只用逻辑回归？

你可以这样回答：

- 数据是中等规模表格数据；
- 结构化特征明显；
- XGBoost 在表格数据上通常表现强；
- 训练效率高；
- 与 SHAP 配合方便；
- 对本科毕设来说，复杂度和效果比较平衡。

## 5.5 局限性要主动写

这反而会加分，因为显得你清醒。

必须主动承认：

- 数据只有二分类标签；
- 没有三类欺诈的真实监督标签；
- 当前三类欺诈分析属于类型倾向分析；
- 数据是地址级聚合数据，不是完整交易图或合约源码数据。

## 6. 答辩时最容易被问的问题

## 6.1 为什么你不直接做三分类

标准回答：

> 当前数据集仅提供正常 / 欺诈二分类标签，缺乏三类欺诈的真实监督标签，因此本文将三类欺诈分析设计为基于模型解释和业务规则的类型倾向分析，而非直接监督三分类任务。

## 6.2 为什么选 XGBoost

标准回答：

> 因为本研究数据属于结构化表格数据，特征维度具有明显业务含义，XGBoost 在这类数据上通常具有较好的性能与稳定性，同时便于与 SHAP 结合开展可解释性分析。

## 6.3 你的模型最重要的字段是什么

你要能回答出这一类：

- 生命周期相关字段
- 收发金额相关字段
- 收款地址数和交易频次相关字段
- ERC20 活跃度相关字段

不要死记一个单独字段，要按“类别”来回答。

## 6.4 你的创新点是什么

本科阶段不要把创新说得太夸张。  
最稳妥的说法是：

1. 将链上欺诈风险识别与三类典型欺诈业务逻辑进行了结合；
2. 在模型预测基础上加入 SHAP 可解释分析；
3. 进一步把模型解释结果用于类型倾向判断，而不是只停留在黑箱预测。

## 6.5 你的不足是什么

你可以主动说：

1. 缺乏三类欺诈的真实标签；
2. 数据仍然是聚合级而不是原始图结构；
3. 文本 token 字段在基线中尚未编码利用；
4. 后续可补充更多外部标签和图结构信息。

如果把整个课题压缩成一句话，就是：

> XGBoost 负责识别欺诈风险，SHAP 负责解释关键特征，三类欺诈分析负责说明高风险地址更接近哪种业务模式。
