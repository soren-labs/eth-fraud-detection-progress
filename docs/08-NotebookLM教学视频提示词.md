# NotebookLM 教学视频提示词

## 1. 使用建议

建议你把下面这些材料一起上传到 NotebookLM：

- `docs/07-XGBoost基础知识补充.md`
- `docs/03-欺诈类型与数据集字段映射.md`
- `docs/02-数据集分析情况.md`
- `processed/transaction_dataset_cleaning_report.md`

然后在 NotebookLM 的自定义提示词中，直接粘贴下面这段内容。

## 2. 推荐提示词

```text
请基于我上传的文档，为“区块链欺诈检测毕业设计”生成一份中文教学视频脚本或教学讲解，目标观众是几乎没有机器学习基础的本科生。请严格围绕当前课题来讲，不要泛泛而谈。

讲解目标：
1. 让学生理解 XGBoost 在本课题中的作用。
2. 让学生理解当前模型做的是“欺诈风险预测”，不是严格的三类欺诈监督分类。
3. 让学生理解训练具体会用到哪些字段，以及这些字段为什么重要。
4. 让学生理解 SHAP 和 LLM 在整个流程中的位置。

讲解风格要求：
- 用中文讲。
- 语气像一位耐心、逻辑清晰的老师，不要像百科全书。
- 面向零基础，默认学生不懂监督学习、决策树、boosting、AUC、F1。
- 少讲公式，多讲直觉、类比、业务含义。
- 不要空讲机器学习发展史，要紧贴当前毕业设计。
- 如果提到英文字段名，请先读英文名，再立刻解释中文含义。
- 如果提到指标，请解释这个指标在“欺诈检测”里为什么重要。
- 多用“在我们这个课题里”这种表述，避免脱离项目。

内容结构要求：
第一部分：先用非常简单的话说明本课题当前到底要解决什么问题。
第二部分：解释什么是监督学习，什么是二分类，什么是标签 FLAG。
第三部分：解释为什么当前数据更适合先做“欺诈风险预测”，而不是直接做三种欺诈的监督分类。
第四部分：解释什么是决策树，再解释什么是 XGBoost，最好用“多个老师接力纠错”这种直观类比。
第五部分：结合当前课题，重点讲清楚训练用到的字段，按下面几组来讲：
1. 时间与生命周期字段
2. 交易频次与网络字段
3. ETH 金额字段
4. ERC20 频次与网络字段
5. ERC20 金额字段

第六部分：一定要结合当前真实字段举例说明，不要只说抽象概念。重点说明这些字段为什么能帮助识别：
1. 钓鱼诈骗与资金清洗
2. 庞氏骗局
3. 发币骗局与跑路盘

第七部分：解释训练流程时，请讲清楚：
- 训练集和测试集是什么
- 为什么要看 Accuracy、Precision、Recall、F1、AUC
- 为什么在欺诈检测里不能只看 Accuracy
- 什么是类别不平衡
- 为什么当前要先用数值字段，不先用 Address 和两个 token 文本字段

第八部分：解释 SHAP 在本课题中的作用。请明确说明：
- SHAP 不是重新训练模型
- SHAP 是解释模型为什么这么判断
- SHAP 可以帮助判断一个高风险地址更接近哪种欺诈模式

第九部分：解释 LLM 在本课题中的作用。请明确说明：
- LLM 不是主分类器
- LLM 的作用是把模型输出和 SHAP 结果翻译成自然语言
- 最终可以生成论文中的案例解释

第十部分：最后帮学生总结一句最重要的话：
“XGBoost 负责判断欺诈风险，SHAP 负责解释关键特征，LLM 负责生成自然语言说明；三类欺诈类型的判断目前更适合被表述为类型倾向分析，而不是严格的监督分类。”

额外要求：
- 讲解时请尽量引用当前课题中的真实字段名，例如：
  - Time Diff between first and last (Mins)
  - Avg min between sent tnx
  - total ether balance
  - Unique Received From Addresses
  - Number of Created Contracts
  - Total ERC20 tnxs
  - ERC20 total Ether received
- 请自然说明这些字段分别和哪种欺诈逻辑更相关。
- 不要默认观众已经学过机器学习。
- 整体节奏适中，适合做 8 到 12 分钟左右的教学视频。
- 结尾请给出一个适合本科毕业设计的完整理解框架，让学生知道后续实验要怎么做。
```

## 3. 如果你想让视频更偏“老师讲课”风格

可以在上面的提示词最后再补一句：

```text
请尽量讲得像导师在给学生补基础课，不要像产品宣传片，也不要像学术论文朗读。
```

## 4. 如果你想让视频更偏“答辩前复习”风格

可以在上面的提示词最后再补一句：

```text
请在每一部分结尾都用一句“你可以记住：……”来帮学生总结答辩时最该说的话。
```

