# XGBoost 基础知识补充

## 1. 这份文档是做什么的

这份文档面向**几乎没有机器学习基础**的读者，目标不是把 XGBoost 讲成一本教材，而是让你能够理解后续毕业设计中的训练计划，知道：

- 我们为什么选 XGBoost；
- XGBoost 在当前课题里到底学的是什么；
- 哪些字段会进入训练；
- 模型输出的“欺诈风险”是什么意思；
- SHAP 和 LLM 在整个流程中分别扮演什么角色。

你可以把这份文档理解为“面向本课题的 XGBoost 入门说明书”。

## 2. 先用一句话理解我们当前的任务

我们现在最稳妥的任务，不是“让模型直接判断它一定是钓鱼、庞氏还是跑路盘”，而是：

1. 让模型先判断一个地址**有没有欺诈风险**；
2. 再结合 SHAP 看这个地址为什么会被判成高风险；
3. 最后结合你定义的三类欺诈规则，分析它**更接近哪一类欺诈**；
4. 让 LLM 把这些结果翻译成自然语言说明。

所以在当前阶段，XGBoost 的核心任务是：  
**根据地址级特征，学习“正常地址”和“欺诈地址”的差异。**

## 3. 当前课题中的“训练”到底在做什么

### 3.1 监督学习的最简单理解

监督学习可以理解成：

- 我们给模型一批“已经知道答案”的样本；
- 每个样本都带着特征和标签；
- 模型通过这些样本学习“什么样的特征组合更像欺诈”；
- 之后遇到一个新地址，模型就能给出风险判断。

在你的课题里：

- **样本**：一个以太坊地址
- **特征**：这个地址的交易行为统计信息
- **标签**：`FLAG`
  - `0` 表示正常地址
  - `1` 表示欺诈地址

### 3.2 训练数据长什么样

经过清洗后，当前建模数据文件是：

- `processed/transaction_dataset_cleaned.csv`

数据基本情况：

- 总样本数：9816
- 总字段数：42
- 标签字段：`FLAG`
- 地址字段：`Address`
- 额外新增字段：`has_erc20_activity`

## 4. 在我们课题里，XGBoost 的输入和输出是什么

### 4.1 输入是什么

XGBoost 训练时输入的是一行行地址特征。  
最简单的基线版本里，我们先用**数值字段**训练，不把文本字段直接喂给模型。

### 4.2 输出是什么

输出通常有两种理解方式：

1. **预测类别**
   - `0`：正常
   - `1`：欺诈

2. **预测概率**
   - 例如 `0.91`
   - 表示模型认为这个地址是欺诈地址的概率很高

在论文里，第二种表述通常更好，因为它更像“风险分数”。

## 5. 当前训练涉及到的具体字段

### 5.1 不进入基线训练的字段

为了先做一个稳定、简单、容易解释的版本，下面这几个字段**先不进入基线训练**：

- `Address`
  - 它是地址标识，不是行为特征；
  - 留着做追踪，不参与训练。

- `ERC20 most sent token type`
- `ERC20_most_rec_token_type`
  - 这两个是文本类别字段；
  - 当前缺失、`0`、空白很多；
  - 初版模型先不用，后续若需要再做编码。

### 5.2 进入基线训练的核心数值字段

清洗后，当前最适合进入基线训练的是 **38 个数值字段**。可以按业务含义分成 5 组理解。

#### A. 时间与辅助指示字段

- `has_erc20_activity`
- `Avg min between sent tnx`
- `Avg min between received tnx`
- `Time Diff between first and last (Mins)`

这一组主要回答：

- 地址活多久？
- 发送钱快不快？
- 收钱快不快？
- 是否有 ERC20 相关活动？

这组字段对钓鱼诈骗和快速洗钱很重要，因为那类地址通常“短、快、集中”。

#### B. 交易频次与网络字段

- `Sent tnx`
- `Received Tnx`
- `Number of Created Contracts`
- `Unique Received From Addresses`
- `Unique Sent To Addresses`
- `total transactions (including tnx to create contract)`

这一组主要回答：

- 它是更偏向收款，还是更偏向付款？
- 它和多少不同地址交互？
- 它有没有创建过合约？

这组字段对庞氏骗局和发币骗局都很重要。

#### C. ETH 金额字段

- `min value received`
- `max value received`
- `avg val received`
- `min val sent`
- `max val sent`
- `avg val sent`
- `min value sent to contract`
- `max val sent to contract`
- `avg value sent to contract`
- `total Ether sent`
- `total ether received`
- `total ether sent contracts`
- `total ether balance`

这一组主要回答：

- 它一次收多少钱、发多少钱？
- 它累计收了多少、发了多少？
- 它向合约转过多少钱？
- 它整体净流量如何？

这组字段和资金归集、返利、出金、吸筹都直接相关。

#### D. ERC20 频次和网络字段

- `Total ERC20 tnxs`
- `ERC20 total Ether received`
- `ERC20 total ether sent`
- `ERC20 total Ether sent contract`
- `ERC20 uniq sent addr`
- `ERC20 uniq rec addr`
- `ERC20 uniq rec contract addr`
- `ERC20 uniq sent token name`
- `ERC20 uniq rec token name`

这一组主要回答：

- 这个地址的代币交易活跃不活跃？
- 它收发了多少代币？
- 它和多少个代币地址、代币合约交互？
- 它涉及多少种代币？

这组字段对“发币骗局与跑路盘”尤其重要。

#### E. ERC20 金额字段

- `ERC20 min val rec`
- `ERC20 max val rec`
- `ERC20 avg val rec`
- `ERC20 min val sent`
- `ERC20 max val sent`
- `ERC20 avg val sent`

这一组主要回答：

- 代币收发的金额规模和波动大不大；
- 是否存在异常代币流转。

## 6. XGBoost 到底是什么

### 6.1 最直观的理解

XGBoost 可以简单理解成：

> 它不是只训练一棵决策树，而是训练很多棵“小树”，让后一棵树不断修正前一棵树犯过的错误，最后把大家的判断综合起来。

你可以把它想象成一组接力老师：

- 第一位老师先做一个粗判断；
- 第二位老师专门纠正第一位老师看错的地方；
- 第三位老师继续修正前两位老师还没看好的部分；
- 最后把所有老师的意见综合成一个结果。

这就是“Boosting（提升）”的核心直觉。

### 6.2 决策树在我们课题里怎么理解

一棵树会不断问类似这样的问题：

- `Time Diff between first and last (Mins)` 是否很小？
- `avg val received` 是否较高？
- `Unique Received From Addresses` 是否很多？
- `Number of Created Contracts` 是否大于 0？
- `Total ERC20 tnxs` 是否明显偏高？

通过一连串“是 / 否”判断，它会把地址一步步分到不同叶子节点，最后输出一个倾向。

### 6.3 为什么 XGBoost 适合我们这个课题

XGBoost 很适合你现在的课题，原因是：

- 你的数据是典型的**表格型数据**；
- 字段之间关系明显不是线性的；
- 有些欺诈行为依赖“字段组合”而不是单个字段；
- 它对数值型特征表现通常很好；
- 训练速度快，适合本科阶段反复试验；
- 后续配合 SHAP，解释性也比较强。

## 7. 训练 XGBoost 时要理解的几个关键概念

### 7.1 训练集和测试集

训练时不能把所有数据都拿来学完再自己给自己打分。  
通常会拆成：

- **训练集**：模型真正学习用的数据
- **测试集**：训练后拿来检查模型效果的数据

这一步是为了看模型对“没见过的新样本”是否还能表现稳定。

### 7.2 类别不平衡

现在清洗后数据里：

- 正常地址：7637
- 欺诈地址：2179

也就是说，正常样本更多。  
这会导致模型天然更容易偏向“猜正常”。

所以训练 XGBoost 时，通常会设置：

- `scale_pos_weight`

这个参数的作用可以简单理解成：

> 提醒模型不要忽视少数类，也就是欺诈地址。

### 7.3 过拟合

过拟合就是：

- 模型把训练集记得太死；
- 在训练集上很好；
- 一到新数据上就变差。

所以我们不会一味追求训练集分数，而更看重测试集分数。

## 8. 你后面会用到的 XGBoost 参数，先记住这几个就够了

### 8.1 `n_estimators`

表示训练多少棵树。

- 太少：模型可能学不够；
- 太多：训练更久，也可能过拟合。

### 8.2 `max_depth`

表示每棵树允许长多深。

- 越深，模型越复杂；
- 越浅，模型越保守。

### 8.3 `learning_rate`

表示每一轮修正错误时走多大一步。

- 大：学得快，但可能不稳定；
- 小：学得慢，但通常更稳。

### 8.4 `subsample`

表示每轮训练时，只随机抽一部分样本来训练树。  
它有助于降低过拟合。

### 8.5 `colsample_bytree`

表示每棵树只随机看一部分字段。  
它也有助于降低过拟合。

### 8.6 `eval_metric`

表示你用什么指标来评估训练过程。  
二分类里常见的有：

- `logloss`
- `auc`

## 9. 在这个课题里，我们更该关注哪些指标

做欺诈检测时，不应该只看 Accuracy。

### 9.1 Accuracy

整体预测对了多少。

### 9.2 Precision

被模型判成欺诈的地址里，有多少真的欺诈。

### 9.3 Recall

真实欺诈地址里，有多少被模型抓住了。

### 9.4 F1

Precision 和 Recall 的综合平衡指标。

### 9.5 AUC

看模型整体区分正常地址和欺诈地址的能力。

对你这个课题来说，建议重点看：

- `F1`
- `Recall`
- `AUC`

因为欺诈检测里“有没有抓住风险地址”很重要。

## 10. 结合当前课题的 XGBoost 基线结果

我已经用清洗后的数据跑过一个简化版 XGBoost 基线，做的是：

- 输入：38 个数值字段
- 不使用：`Address`、两个 token 文本字段

- Accuracy = 0.9588

这说明：

- XGBoost 在你当前数据上是完全可以训练的；
- 而且结果已经达到一个本科毕设完全可接受、甚至算比较好的水平。

当前 XGBoost 基线里，重要性较高的字段包括：

- `total ether received`
- `avg val received`
- `Time Diff between first and last (Mins)`
- `ERC20 min val rec`
- `Avg min between received tnx`
- `Unique Received From Addresses`
- `total transactions (including tnx to create contract)`
- `has_erc20_activity`
- `total Ether sent`
- `total ether balance`

这组结果其实非常符合你当前课题的业务理解：

- **钓鱼诈骗与资金清洗**：短生命周期、快速收发、净余额低；
- **庞氏骗局**：收款多、入金地址多、平均收款金额有模式；
- **发币骗局与跑路盘**：ERC20 活跃、代币接收明显、合约行为相关。

## 12. SHAP 在这里是做什么的

SHAP 的核心作用不是重新训练模型，而是解释模型：

- 为什么这个地址被判成高风险？
- 是哪几个字段把它推向欺诈？
- 每个字段贡献了多少？

### 12.1 全局解释

看整个模型整体上最依赖哪些字段。  
例如：

- 生命周期
- 收款金额
- ERC20 活跃度

### 12.2 局部解释

看某一个具体地址为什么被判成高风险。  
例如某个地址：

- `Time Diff between first and last (Mins)` 非常短；
- `total ether balance` 很低；
- `Avg min between sent tnx` 非常短；

那么它就更像“钓鱼诈骗与资金快速转移”。

## 13. LLM 在这里是做什么的

LLM 不负责替代 XGBoost 训练。  
更合适的分工是：

- XGBoost：负责预测风险
- SHAP：负责解释重要字段
- LLM：负责把这些结果翻译成自然语言

一个典型输出可以写成：

> 该地址被模型判定为高风险欺诈地址。主要原因是其生命周期较短、平均发送时间间隔极短、净余额较低，并存在明显的资金快速流转行为，因此更接近钓鱼诈骗与资金清洗的行为模式。

这类输出很适合直接写进论文案例分析部分。

## 14. 你需要避免的一个误区

当前模型学到的是：

- **欺诈风险**

它不是直接从数据标签里学到：

- 钓鱼
- 庞氏
- 跑路盘

因为你当前数据只有 `FLAG`，没有三种欺诈类型的真实标签。

所以后续更严谨的表述应该是：

- XGBoost 负责预测欺诈风险；
- SHAP 帮助识别该地址的关键风险特征；
- 再结合三类欺诈的业务规则，分析其更接近哪一类欺诈；
- LLM 生成可解释说明。

## 15. 后续训练计划，你现在应该怎么理解

如果按最适合本科毕设的路线推进，后续训练流程可以理解成 4 步：

1. **用清洗后的数值字段训练 XGBoost 二分类模型**
2. **用 Accuracy / Recall / F1 / AUC 检查模型效果**
3. **用 SHAP 做全局解释和单地址解释**
4. **结合三类欺诈规则 + LLM 输出自然语言分析**

所以你不用一开始就把 XGBoost 想得很复杂。  
在这个课题里，它本质上只是一个：

> 利用地址行为字段来学习“像不像欺诈地址”的强力表格模型。

## 16. 你现在至少要记住的 6 句话

1. XGBoost 是很多小树不断纠错形成的模型。
2. 我们现在做的是“欺诈风险预测”，不是严格的三分类监督学习。
3. 当前基线训练先用数值字段，不用 `Address` 和两个 token 文本字段。
4. 重要字段主要来自时间、收付款规模、交互地址数和 ERC20 活跃度。
5. SHAP 负责解释模型为什么这么判断。
6. LLM 负责把模型结果和 SHAP 结果讲成人能看懂的话。
