# 论文整体章节设计

## 1. 设计原则

本章节设计以你的开题报告 `开题报告_郑壹夫_最终.docx` 为主线，保留其总体结构，但结合你当前目录中已经完成的工作进行细化。  
当前已经具备的材料包括：

- 三类欺诈类型分析
- 数据集分析与字段映射
- 数据清洗方案与清洗结果
- 随机森林基线实验
- XGBoost 基线与优化实验
- SHAP 全局解释与单地址案例解释

因此，正式论文不应该再写得很虚，而应该尽量围绕“**已有数据、已有模型、已有实验结果**”展开。

## 2. 总体章节建议

建议正式论文采用以下 8 章结构：

1. 绪论
2. 相关理论与技术基础
3. 研究对象与欺诈类型分析
4. 数据集构建与特征工程
5. 模型构建与实验设计
6. 实验结果与模型优化分析
7. 可解释性分析与欺诈类型倾向判断
8. 总结与展望

后面再接：

- 参考文献
- 致谢

这个结构相比开题报告更具体，且能够自然容纳你已经完成的全部工作。

## 3. 各章详细设计

## 第 1 章 绪论

### 本章目标

让老师一开始就明白：

- 你研究的是什么问题
- 为什么要做这个问题
- 你用什么方法做
- 你的论文到底完成了什么

### 建议小节

#### 1.1 研究背景

重点写：

- 区块链与以太坊生态的发展
- 链上欺诈问题越来越突出
- 传统黑名单和人工分析存在不足
- 欺诈检测需要兼顾准确性和可解释性

#### 1.2 研究意义

分成两部分写：

- **理论意义**：将机器学习、可解释性分析与 LLM 辅助说明结合
- **实践意义**：提高欺诈地址识别透明度，帮助交易所、监管方、普通用户理解风险

#### 1.3 国内外研究现状

建议分 3 类写：

1. 链上欺诈检测研究现状
2. 区块链欺诈类型分析研究现状
3. 可解释性方法与 LLM 辅助解释研究现状

这里可以引用：

- `docs/欺诈类型报告-中文版.md`
- 开题报告里的参考文献

#### 1.4 研究内容与技术路线

这一节非常重要，建议直接概括成：

1. 对三类欺诈类型进行业务特征分析
2. 对公开数据集进行清洗和字段映射
3. 构建随机森林与 XGBoost 模型进行欺诈风险识别
4. 通过 SHAP 完成全局与局部解释
5. 结合三类欺诈规则完成类型倾向判断

#### 1.5 论文创新点与主要工作

本科阶段不要写得太夸张，建议这样写：

1. 将三类典型欺诈业务逻辑与地址级交易特征建立映射
2. 构建“基线模型—主模型—优化模型”的完整实验链条
3. 在欺诈风险识别基础上加入 SHAP 可解释性分析
4. 进一步将模型解释结果用于三类欺诈倾向判断

#### 1.6 论文结构安排

这里就用一句话概括每一章做什么。

## 第 2 章 相关理论与技术基础

### 本章目标

给后面的方法和实验做理论铺垫，避免论文一上来就直接跑模型。

### 建议小节

#### 2.1 区块链与以太坊交易基础

建议写：

- 区块链基本概念
- 以太坊账户与交易
- 地址级交易行为特征的含义
- ETH 与 ERC20 的区别

#### 2.2 链上欺诈检测的基本思路

建议写：

- 地址行为特征识别
- 规则分析
- 机器学习分类
- 可解释性分析

#### 2.3 树模型与集成学习基础

重点写：

- 决策树原理
- 随机森林原理
- Boosting 思想
- XGBoost 原理与优势

这一节要尽量写得通俗，因为后面答辩很可能问你：

- 为什么要用 XGBoost
- XGBoost 比随机森林强在哪里

#### 2.4 模型可解释性方法

重点写：

- 可解释性分析的必要性
- 全局解释与局部解释
- SHAP 方法的基本思想

#### 2.5 LLM 在可解释性输出中的作用

这一节建议写得克制一点，不要把 LLM 写成主分类器。  
应强调：

- LLM 不是用来替代模型训练
- LLM 更适合做“解释结果的自然语言组织”
- 在本课题中，LLM 的定位是辅助解释层

## 第 4 章 数据集构建与特征工程

### 本章目标

把“数据从哪里来、清洗了什么、最终拿什么训练”讲清楚。

### 建议小节

#### 4.1 数据集来源与标签定义

建议写：

- 数据集来源
- 样本规模
- 标签字段 `FLAG`
- 为什么当前任务本质是二分类

#### 4.2 原始数据集问题分析

建议写：

- 缺失值情况
- 重复地址
- 列名不规范
- 文本字段编码混乱
- 恒定列与低信息量列

#### 4.3 数据清洗流程

这一节应当非常具体，写：

- 删除哪些字段
- 为什么删
- 如何处理重复地址
- 如何处理 ERC20 缺失
- 如何处理 token 文本字段
- 为什么保留 `total ether balance`

#### 4.4 特征体系构建

建议按 5 大类整理：

1. 时间与生命周期特征
2. 交易频次与网络特征
3. ETH 金额特征
4. 合约交互特征
5. ERC20 特征

#### 4.5 训练特征确定

写：

- 为什么基线模型只用数值字段
- 为什么不使用 `Address`
- 为什么两个 token 文本字段先不直接进入基线训练
- 优化时如何引入频次编码

## 第 5 章 模型构建与实验设计

### 本章目标

把整篇论文收住，不要只写套话。

### 建议小节

#### 8.1 工作总结

建议按“完成了什么”来写：

1. 完成三类欺诈类型分析
2. 完成数据清洗与字段映射
3. 完成基线模型与主模型构建
4. 完成 XGBoost 优化实验
5. 完成 SHAP 可解释性分析

- 引入更多外部标签数据

## 5. 最终建议

如果你希望论文写得稳，建议坚持以下原则：

1. 主线必须明确：**欺诈风险识别 + 类型倾向分析**
2. 不要硬写成真实三分类监督学习
3. 重点突出：**XGBoost 优化过程 + SHAP 解释**
4. LLM 部分可以作为增强项，不要喧宾夺主

如果要把整篇论文压缩成一句话，那就是：
