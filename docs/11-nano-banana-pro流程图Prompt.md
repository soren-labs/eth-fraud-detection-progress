## 2. 推荐 Prompt

下面这段可以直接用于 `nano banana pro` 生成论文汇报流程图。

```text
请生成一张“本科毕业设计研究流程图 / 学术技术路线图”，风格要偏学术论文配图，而不是商业宣传海报。要求画面干净、结构清晰、文字准确、排版规整、适合直接放进毕业论文或中期汇报 PPT。

整体要求：
- 画幅比例：16:9 横版
- 风格：学术信息图、论文插图风格、简洁矢量、白底、蓝绿灰配色、轻微科技感
- 不要卡通风，不要 3D，不要夸张光效，不要复杂背景
- 所有文字使用简体中文，字体统一，标题清晰，文字对齐整齐
- 重点强调“研究流程”和“已完成 / 待完成”的阶段划分
- 流程箭头明确，从左到右或从上到下，逻辑不能混乱
- 请尽量保证图中文字清晰、准确、易读

图的总标题请写为：
“基于 ML 与 LLM 的可解释性以太坊欺诈检测研究流程”

副标题请写为：
“本科毕业设计技术路线图（当前进度版）”

整体结构请设计成 9 个主模块的流程图，每个模块用矩形或圆角矩形表示，模块之间用箭头连接，排版为学术型流程图。

9 个主模块及模块标题和说明文字如下，请尽量逐字准确呈现：

1. 研究问题定义
说明文字：
“目标：构建以太坊地址欺诈风险识别模型，并结合可解释性分析进行欺诈类型倾向判断”

2. 欺诈类型分析
说明文字：
“重点研究三类欺诈：
1）钓鱼诈骗与资金清洗
2）庞氏骗局
3）发币骗局与跑路盘”

3. 数据集构建与清洗
说明文字：
“公开数据集整理与预处理：
- 字段统一
- 缺失值处理
- 重复地址合并
- ERC20 特征规范化”

4. 特征体系构建
说明文字：
“构建 5 类特征：
- 时间与生命周期特征
- 交易频次与网络特征
- ETH 金额特征
- 合约交互特征
- ERC20 代币特征”

5. 基线模型训练
说明文字：
“随机森林基线模型
用于建立可比较的初始实验结果”

6. 主模型训练与优化
说明文字：
“XGBoost 主模型
优化方式包括：
- 超参数调整
- 文本频次编码
- 轻量特征工程
- 阈值优化”

7. 模型评估与对比
说明文字：
“评价指标：
- Accuracy
- Precision
- Recall
- F1-score
- ROC-AUC

最终推荐主模型：
优化后的 XGBoost”

8. SHAP 可解释性分析
说明文字：
“解释层面包括：
- 全局特征重要性
- 单地址案例解释
- 欺诈类型倾向分析”

9. LLM 辅助解释生成（待完成）
说明文字：
“基于模型结果与 SHAP 特征贡献，
生成自然语言风险说明与案例描述
当前阶段：待完成”

设计要求补充：
- 前 8 个模块使用正常实线边框
- 第 9 个模块“LLM 辅助解释生成（待完成）”使用灰色虚线边框或浅灰色模块，明显表示这是尚未完成的阶段
- 在图中加入一个小型进度提示标识，文字为：
“当前已完成：数据清洗、基线模型、XGBoost 优化、SHAP 解释”
- 在图中加入一个小型成果提示标识，文字为：
“当前最优结果：
F1 = 0.9306
ROC-AUC = 0.9923”

视觉层级要求：
- 总标题最大
- 模块标题次之
- 模块说明文字比标题略小
- 文字块不要过密，每个模块保持清晰留白
- 箭头和模块边框要整齐统一

请输出为一张高质量、适合论文和 PPT 使用的学术流程图，整体观感要像“毕业论文中的技术路线图”。
```

## 3. 如果你想让图更像“论文图 1”

可以在 prompt 最后再补一句：

```text
请让整张图更接近中文本科毕业论文中的“图1 技术路线图”风格，减少装饰感，强调结构、逻辑和文字准确性。
```

## 4. 如果你想让图更像“中期汇报页”

可以在 prompt 最后再补一句：

```text
请增强“阶段进度”表达，让已完成部分更清晰，未完成的 LLM 阶段更弱化但仍保留在流程末端。
```
