# XGBoost 优化结果汇总

## 1. 优化目标

在已经获得可用基线结果的前提下，进一步通过多种优化方式提升 XGBoost 在欺诈检测任务中的表现，并形成可写入论文的“优化过程”。

本轮优化保持以下实验条件不变：

- 使用同一份清洗后数据：`transaction_dataset_cleaned.csv`
- 保持固定随机种子：`42`
- 保持固定测试集划分
- 仅在训练集和验证集上进行参数与策略搜索

## 2. 本轮实际尝试的优化方式

本轮共运行 **32 个实验组合**，主要尝试了以下 4 类优化方法：

1. **超参数优化**
   - 调整 `n_estimators`
   - 调整 `max_depth`
   - 调整 `learning_rate`
   - 调整 `subsample`
   - 调整 `colsample_bytree`
   - 调整 `min_child_weight`
   - 调整 `gamma`
   - 调整 `reg_alpha`
   - 调整 `reg_lambda`
   - 调整 `scale_pos_weight`

2. **轻量特征工程**
   - 构造收发笔数比值
   - 构造收发金额比值
   - 构造唯一地址比值
   - 构造 ERC20 活跃密度等衍生特征

3. **文本字段频次编码**
   - 对 `ERC20 most sent token type`
   - 对 `ERC20_most_rec_token_type`
   进行频次编码，而不是直接丢弃

4. **阈值优化**
   - 不只使用默认阈值 `0.5`
   - 在验证集上搜索更优分类阈值

## 3. 最优实验方案

验证集上表现最优的方案为：

- 实验名称：`deeper_more_trees__numeric_textfreq`
- 特征方案：`numeric_textfreq`
- 最优验证阈值：`0.66`

对应参数为：

- `n_estimators = 500`
- `max_depth = 5`
- `learning_rate = 0.04`
- `subsample = 0.8`
- `colsample_bytree = 0.8`
- `min_child_weight = 1`
- `gamma = 0.0`
- `reg_alpha = 0.0`
- `reg_lambda = 1.0`

这一结果说明：

- 更深一点的树
- 更多的树
- 较低的学习率
- 加入 token 文本频次编码

对于当前任务是有效的。

## 4. 优化后的测试集结果

### 4.1 默认阈值 0.5

优化后模型在测试集上的结果为：

- Accuracy = `0.9695`
- Precision = `0.9393`
- Recall = `0.9220`
- F1-score = `0.9306`
- ROC-AUC = `0.9923`
- Average Precision = `0.9799`

### 4.2 验证集调优阈值 0.66

使用验证集选出的最优阈值 `0.66` 后，测试集结果为：

- Accuracy = `0.9700`
- Precision = `0.9631`
- Recall = `0.8991`
- F1-score = `0.9300`
- ROC-AUC = `0.9923`
- Average Precision = `0.9799`

## 5. 哪个结果最终更推荐

虽然验证集阈值调优在搜索阶段是有效的，但在测试集上：

- 默认阈值 `0.5` 的 **F1-score 更高**
- 默认阈值 `0.5` 的 **Recall 更高**

因此，当前更推荐的最终模型是：

> **优化后的 XGBoost + 默认阈值 0.5**

这也是本次优化中一个很有价值的结论：

> 阈值优化是值得尝试的，但不一定在独立测试集上带来更好的综合表现。

这句话很适合写进论文“实验分析”部分。

## 6. 与基线 XGBoost 的对比

原始 XGBoost 基线结果为：

- Accuracy = `0.9588`
- Precision = `0.8918`
- Recall = `0.9266`
- F1-score = `0.9089`
- ROC-AUC = `0.9906`
- Average Precision = `0.9758`

优化后推荐结果为：

- Accuracy = `0.9695`
- Precision = `0.9393`
- Recall = `0.9220`
- F1-score = `0.9306`
- ROC-AUC = `0.9923`
- Average Precision = `0.9799`

### 提升情况

- Accuracy：`+0.0107`
- Precision：`+0.0474`
- Recall：`-0.0046`
- F1-score：`+0.0217`
- ROC-AUC：`+0.0016`
- Average Precision：`+0.0040`

### 结果解读

可以看出：

- 召回率略有下降，但幅度很小；
- 精确率提升明显；
- F1-score 提升明显；
- AUC 和 AP 也有提升。

因此，从综合平衡上看，优化后的 XGBoost 比基线更强。

## 7. 与随机森林基线的关系

随机森林基线结果为：

- Accuracy = `0.9603`
- Precision = `0.9686`
- Recall = `0.8486`
- F1-score = `0.9046`
- ROC-AUC = `0.9863`

对比来看：

- 优化后的 XGBoost 仍然是当前最适合作为主模型的方案；
- 随机森林可以保留为基线对照模型；
- XGBoost 更适合后续开展 SHAP 可解释分析。

## 8. 论文中推荐的写法

可以直接写成：

> 在基线模型基础上，本文进一步从超参数调整、轻量特征工程、文本频次编码和分类阈值优化四个方面对 XGBoost 模型进行了系统优化，共比较 32 组实验组合。结果表明，加入 token 频次编码并适当增加树深与树数量后，模型整体性能得到提升。最终采用的优化方案在测试集上取得了 0.9306 的 F1-score 和 0.9923 的 ROC-AUC，较基线模型有明显改善。

如果需要强调实验过程，也可以再补一句：
